# 01_discovery_call_guide.md

## Discovery sections
1. Business priorities
2. Current workflow and pain points
3. Systems/data/integrations
4. Stakeholders and approval flow
5. Success metrics / ROI
6. Constraints (timeline/budget/compliance/security)
7. Delivery preferences
8. Risks/unknowns
9. Prioritization (must/should/nice)
10. Proceed/follow-up/blocked qualification

## Targeted questions for this project

### A) Information architecture
1. What are the must-have sections for MVP? (e.g., About, Work, Projects, Writing, Contact)
2. Which sections are must-have vs nice-to-have for one-day build?
3. Do you want chronological resume style, narrative style, or case-study style?

### B) Format decision (Web vs TUI vs Hybrid)
1. Preferred default entry point: standard web URL or terminal experience?
2. If hybrid: what should be visible on public web page vs terminal mode?
3. What is acceptable accessibility tradeoff for a terminal-first experience?
4. Share the reference TUI website link for pattern analysis.

### C) Brand and design
1. 2–3 reference sites you like (especially PM/tech profiles).
2. Desired vibe: minimal, bold, playful, technical, premium?
3. Any color/typography preferences or existing personal brand assets?

### D) Content readiness
1. Do you already have: short bio, long bio, project summaries, metrics, testimonials?
2. Which projects should be highlighted in MVP (top 3)?
3. What contact channels should be shown? (email, LinkedIn, X, GitHub)

### E) Tech and delivery constraints
1. Preferred stack (Next.js/static/Webflow/etc.)?
2. Hosting preference (Vercel, Netlify, custom)?
3. Any analytics/tracking requirement for MVP?

### F) Success definition
1. What does “impressed” mean in measurable terms? (e.g., recruiter response rate, qualitative feedback)
2. MVP launch deadline (exact date/time)?
